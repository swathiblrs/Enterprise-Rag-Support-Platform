# Ticket Priority Matrix

## Overview
This document defines priority levels for enterprise support tickets.

## Low Priority
Use Low priority when the issue has a workaround and does not block the user from working.

Examples:
- General software questions
- Minor UI issues
- Non-urgent access requests

## Medium Priority
Use Medium priority when one user is blocked from completing normal work but there is no company-wide impact.

Examples:
- One user cannot access VPN
- One user cannot log in to email
- One user needs password reset
- One user cannot approve Duo MFA

## High Priority
Use High priority when multiple users are affected or a business-critical team is blocked.

Examples:
- Multiple users cannot access VPN
- Department-wide email access issue
- Shared application outage

## Critical Priority
Use Critical priority when there is a production outage, security incident, or company-wide service disruption.

Examples:
- Production system outage
- Security breach
- Company-wide authentication failure
