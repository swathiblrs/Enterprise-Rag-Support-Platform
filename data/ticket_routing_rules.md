# Ticket Routing Rules

## Overview
This document defines routing rules for common enterprise IT support tickets.

## Routing Rules
Password reset issues should be routed to Identity and Access Management.

Account lockout issues should be routed to Identity and Access Management.

Duo MFA issues should be routed to Identity and Access Management.

VPN connectivity issues should be routed to Network Support.

Wi-Fi connectivity issues should be routed to Network Support.

Email access issues should be routed to Messaging Support.

Software installation requests should be routed to Desktop Support.

Laptop hardware issues should be routed to Device Support.

Printer issues should be routed to Desktop Support.

Production outage issues should be routed to Site Reliability Engineering.

## Routing Notes
If an issue affects only one user, assign normal priority unless the user is blocked from critical work.
If an issue affects multiple users, increase the priority and notify the support manager.
